Prompting und strukturierte Anweisungen

Die sechs Bausteine eines starken Prompts, XML-Tags, Few-Shot, Chain of Thought, Prompt Chaining, Self-Reflection, Pre-filling und System Prompts — mit konkreten Beispielen aus dem M&A-Alltag.

35 Min. Lernzeit 10 Quiz-Fragen Mit Video

1. Prompt Engineering als iterative Wissenschaft

Prompt Engineering ist kein einmaliges Eintippen, sondern eine empirische Wissenschaft.

  • Der Prozess: Man startet mit einer Basis-Anweisung, analysiert die Fehler der KI und baut den Prompt schrittweise (iterativ) aus, bis das Ergebnis perfekt ist.
  • Das Ziel: Klare Anweisungen und Kontext so strukturieren, dass die KI im ersten Versuch („One-Shot“) das gewünschte Ergebnis liefert.

2. Die 6 Bausteine des perfekten Prompts

Warum Stoppkriterien?

Ohne klare Grenzen „ratet“ Claude lieber, als zuzugeben, dass er unsicher ist. Stoppkriterien wie „Verwende keine Fachbegriffe“ verhindern Halluzinationen.

3. Vom schwachen zum exzellenten Prompt — ein Vergleich

Derselbe Informationsbedarf, drei Qualitätsstufen. Das Beispiel stammt aus dem M&A-Alltag: Sie brauchen ein erstes Profil zu einem möglichen Target.

❌ Schwacher Prompt

„Erzähl mir was über die Firma XY."

Problem: Keine Rolle, keine Struktur, kein Format, keine Grenzen. Claude rät, was gemeint sein könnte, liefert einen beliebigen Fließtext und erfindet im Zweifel Kennzahlen. Das Ergebnis ist unbrauchbar und muss komplett überarbeitet werden.

➖ Guter Prompt

„Du bist M&A-Analyst. Erstelle ein kurzes Company Profile zu [Firma XY]: Geschäftsmodell, Marktposition und mögliche Käufergründe. Antworte als strukturierte Liste."

Besser: Rolle, Aufgabe und Output-Format sind definiert. Das Ergebnis ist brauchbar — aber es fehlen Stoppkriterien gegen Halluzinationen und ein klarer Umfang.

✅ Sehr guter Prompt

„Du bist ein erfahrener M&A-Analyst. Erstelle ein strukturiertes Company Profile zu [Firma XY] mit genau diesen Abschnitten: (1) Geschäftsmodell & Umsatzquellen, (2) Marktposition & zwei bis drei Wettbewerber, (3) mögliche Akquisitionsgründe aus Käuferperspektive. Format: Tabelle plus je ein erläuternder Satz. Maximal 200 Wörter. Wenn Informationen nicht öffentlich verfügbar sind, kennzeichne das explizit mit 'nicht öffentlich verfügbar' — keine Schätzungen, keine erfundenen Zahlen."

Exzellent: Rolle, klar gegliederte Aufgabe, festes Format, Umfangsgrenze und — entscheidend — eine Stoppklausel gegen Halluzinationen. Das Ergebnis ist sofort verwendbar und überprüfbar.

4. Strukturierung durch XML-Tags

Anthropic empfiehlt XML-Tags (z.B. <anweisungen>, <kontext>, <beispiel>), um Informationen sauber zu trennen.

  • Vorteil: Claude wurde darauf trainiert, diese Struktur zu verstehen.
  • Output-Kontrolle: Endergebnis in Tags wie <urteil> setzen für automatisierte Weiterverarbeitung.

5. Fortgeschrittene Prompt-Techniken

Few-Shot Prompting

Zeige Claude 1 bis 3 Beispiele — oft mächtiger als jede lange Beschreibung.

Chain of Thought (Gedankenkette)

Weise Claude an, „Schritt für Schritt zu denken“ — verbessert die Logik massiv.

Prompt Chaining

Riesige Aufgabe in kleine, aufeinanderfolgende Prompts zerlegen. Jedes Teilergebnis prüfen.

Self-Reflection

Claude bitten, seine Antwort selbst auf Fehler zu prüfen und zu überarbeiten.

Pre-filling

Anfang der Antwort vorgeben (z.B. { für JSON) — erzwingt das Format strikt.

6. Effizienz durch System-Prompts & Caching

  • System-Prompts: Statische Informationen (z.B. Firmenformulare) auslagern.
  • Prompt Caching: Spart Zeit, da Claude statische Dokumente nicht neu einlesen muss.

7. Die Reihenfolge der Analyse

Immer Form vor Sketch — erst klare Fakten, dann interpretative Daten.

Merke

Strukturierte Daten liefern den Kontext, in dem unstrukturierte Daten korrekt interpretiert werden.

Praxisübung

Das Gelernte anwenden

Verbessern Sie einen schwachen Prompt systematisch und zerlegen Sie eine komplexe Aufgabe in kontrollierbare Teilschritte.

Übung 1

Schlechten Prompt professionell umbauen

Ein vager Prompt liefert vage Ergebnisse. Überarbeiten Sie den folgenden Prompt systematisch mit den 6 Bausteinen aus diesem Modul.

  • Ausgangsprompt: „Schreib mir was über ein Unternehmen."
  • Ergänzen Sie: Rolle (wer soll Claude sein?), Aufgabe (konkretes Verb), Kontext (Branche, Zweck), Vorgehen (Schrittfolge), Stoppkriterien (keine Vermutungen) und Output-Format (Tabelle, Fließtext, Wortlimit).
  • Testen Sie beide Versionen in Claude und vergleichen Sie die Ergebnisqualität.
  • Dokumentieren Sie die drei wichtigsten Unterschiede.
Beispiel-Prompt (nach Überarbeitung):

Du bist ein M&A-Analyst. Erstelle ein Kurzprofil eines mittelständischen Unternehmens im Bereich Industrieautomation. Struktur: (1) Geschäftsmodell in 2 Sätzen, (2) geschätzte Umsatzgröße, (3) 3 relevante Wettbewerber. Max. 150 Wörter. Wenn Daten nicht verfügbar sind, kennzeichne das explizit.

Ziel: Promptqualität durch systematische Struktur verbessern — vom vagen Input zum präzisen Output.
Übung 2

Prompt Chaining für Analyseaufgaben

Komplexe Aufgaben werden kontrollierbar, wenn man sie in mehrere aufeinander aufbauende Prompts zerlegt. Üben Sie das anhand einer typischen Analyse.

  • Prompt 1: Erstellen Sie ein Kurzprofil für ein Unternehmen Ihrer Wahl (Geschäftsmodell, Marktposition, Kennzahlen).
  • Prompt 2: Lassen Sie dasselbe für zwei weitere Unternehmen derselben Branche erstellen.
  • Prompt 3: Fassen Sie alle drei Profile in einer Vergleichstabelle zusammen, sortiert nach strategischer Passung.
  • Prompt 4: Erstellen Sie eine Management Summary (max. 100 Wörter), die das Ergebnis zusammenfasst.
  • Prüfen Sie nach jedem Schritt: Ist das Zwischenergebnis korrekt und vollständig?
Beispiel-Prompt (Schritt 3):

Fasse die Profile von Unternehmen A, B und C in einer Vergleichstabelle zusammen. Spalten: Unternehmen, Umsatz, Mitarbeiter, Marktposition, strategische Passung (1–5 mit Begründung). Sortiere nach Passung absteigend.

Ziel: Komplexe Aufgaben durch Prompt Chaining kontrollierbar machen — jedes Teilergebnis ist prüfbar.
