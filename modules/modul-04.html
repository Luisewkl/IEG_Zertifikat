Skills und wiederverwendbare Arbeitsabläufe

Einführung in Skills als strukturierte, wiederverwendbare Arbeitsanweisungen — von der inhaltlichen Vorbereitung bis zur praktischen Anwendung im Arbeitsalltag.

Ein Skill ist eine fest definierte, wiederverwendbare Arbeitsanweisung.

Im Skill Creator Modus stellt Claude Rückfragen: Ziel, Zielgruppe, Output-Format, Sprache.

4. Den Skill im Alltag anwenden

  • Über das Plus-Symbol im Chat auswählen.
  • Geringer Prompt-Aufwand dank verankertem Expertenwissen.

5. Fertige Skills aus der AI Template Library nutzen

Neben selbst erstellten Skills gibt es eine wachsende Bibliothek fertiger Vorlagen auf aitmpl.com — die AI Template Library. Hier findest du sofort einsetzbare Skills für verschiedene Anwendungsfälle.

Anleitung: Skill von aitmpl.com herunterladen und einsetzen

  1. Website öffnen: Gehe auf aitmpl.com und durchsuche die verfügbaren Templates nach Kategorie oder Stichwort.
  2. Skill auswählen: Klicke auf ein Template, das zu deinem Anwendungsfall passt (z. B. „Email Writer“, „Meeting Notes“ oder „Research Summary“).
  3. Skill-Text kopieren: Kopiere den vollständigen Skill-Text über den Copy-Button auf der Template-Seite.
  4. In Claude einfügen: Öffne Claude → klicke auf das Plus-Symbol (+) im Chat → wähle „Create a Skill“ → füge den kopierten Text ein.
  5. Anpassen: Passe den Skill bei Bedarf an euren IEG-Kontext an — z. B. Sprache auf Deutsch setzen, Branche auf M&A/Investment Banking spezifizieren, oder Output-Format anpassen.
  6. Nutzen: Der Skill steht ab sofort in jedem neuen Chat über das Plus-Symbol zur Verfügung.
Tipp

Kombiniere fertige Templates von aitmpl.com mit eigenen Anpassungen — so hast du schnell einen professionellen Skill, der genau auf eure M&A-Workflows zugeschnitten ist.

6. Skalierbarkeit

Nicht auf Landing Pages beschränkt — einsetzbar für E-Mail-Funnels, Content-Strategien, Unternehmensstrategien.

Praxisübung

Das Gelernte anwenden

Entwerfen Sie einen eigenen Skill-Blueprint und lernen Sie, wann ein Skill sinnvoller ist als ein einzelner Prompt.

Übung 1

Skill-Blueprint entwerfen

Entwerfen Sie einen einfachen Skill für eine wiederkehrende Aufgabe aus Ihrem Arbeitsalltag — strukturiert und sofort umsetzbar.

  • Wählen Sie eine Aufgabe, die Sie mindestens wöchentlich ausführen (z.B. Gesprächsnotizen aufbereiten, E-Mail-Entwürfe formulieren, Branchenübersichten erstellen).
  • Füllen Sie folgende Felder aus: Name des Skills, Zweck (1 Satz), erwarteter Input, gewünschter Output (Format, Umfang), Regeln (Ton, Sprache, Einschränkungen).
  • Formulieren Sie daraus eine vollständige Skill-Anweisung, die Sie in Claude hinterlegen könnten.
  • Testen Sie den Skill mit einem realistischen Input.
Beispiel-Prompt:

Hilf mir, einen Claude-Skill zu entwerfen. Aufgabe: Aus rohen Gesprächsnotizen einen strukturierten CRM-Eintrag erstellen. Der Skill soll folgende Felder definieren: Name, Zweck, Input-Format, Output-Format, Regeln. Formuliere die vollständige Skill-Anweisung.

Ziel: Wiederverwendbarkeit verstehen — einmal definieren, dauerhaft nutzen.
Übung 2

Skill oder Standard-Prompt?

Nicht jede Aufgabe rechtfertigt einen eigenen Skill. Lernen Sie, die richtige Abstraktionsebene zu wählen.

  • Bewerten Sie folgende drei Aufgaben: (a) Einmalige Zusammenfassung eines Artikels, (b) Wöchentliche Erstellung von Meeting-Protokollen im immer gleichen Format, (c) Gelegentliche Übersetzung einer E-Mail ins Englische.
  • Entscheiden Sie jeweils: Reicht ein einmaliger Prompt oder ist ein gespeicherter Skill sinnvoller?
  • Begründen Sie Ihre Entscheidung anhand der Kriterien: Häufigkeit, Konsistenzanforderung, Komplexität der Anweisung.
Beispiel-Prompt:

Ich gebe dir 3 Aufgaben. Entscheide jeweils, ob ein einmaliger Prompt reicht oder ein gespeicherter Skill sinnvoller wäre. Begründe anhand von Häufigkeit, Konsistenzanforderung und Komplexität. Format: Tabelle mit Spalten Aufgabe, Empfehlung, Begründung.

Ziel: Die richtige Abstraktionsebene wählen — nicht alles braucht einen Skill, aber wiederkehrende Prozesse profitieren enorm.
Musterlösung
AufgabeEmpfehlungBegründung
(a) Einmalige Zusammenfassung eines ArtikelsStandard-PromptNiedrige Häufigkeit, kein Konsistenzbedarf — ein Prompt reicht vollständig
(b) Wöchentliche Meeting-Protokolle im gleichen FormatSkillHohe Häufigkeit + festes Format = klassischer Skill-Kandidat; einmal definieren, dauerhaft nutzen
(c) Gelegentliche Übersetzung einer E-MailStandard-PromptGeringe Häufigkeit, variabler Kontext — ein Skill würde kaum Zeit sparen

Faustregel: Ein Skill lohnt sich ab ca. 3–5 Wiederholungen pro Monat und wenn das gewünschte Format immer gleich ist.
